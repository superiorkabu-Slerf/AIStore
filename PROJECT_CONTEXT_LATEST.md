# AIStore 项目上下文（本线程补充版）

> 更新时间：2026-03-27  
> 说明：此文档是对 `PROJECT_CONTEXT.md` 的补充与对齐，不覆盖原文档。优先记录“当前代码已落地行为”和“后续继续迭代时必须遵守的约束”。

## 1. 项目定位与信息架构

- 项目名称：`AIStore 资讯门户`
- 形态：React + TypeScript 单页应用（Hash Router）
- 核心目标：统一承载 AI 快讯、深度内容、教程、术语、视频、播客、学习路径与工具推荐。

一级分类（固定 6 类）：

- `AI快讯` (`flash`)
- `深度好文` (`article`)
- `精选教程` (`tutorial`)
- `术语百科` (`knowledge`)
- `优质视频` (`video`)
- `AI播客` (`podcast`)

## 2. 技术栈与运行

- React 19 + TypeScript + Vite 6
- 样式：Tailwind 工具类 + 局部全局样式
- 动效：`motion/react`
- 图标：`lucide-react`

常用命令：

- `npm run dev`：本地开发（默认 `3000` 端口）
- `npm run build`：生产构建
- `npm run lint`：TS 类型检查

## 3. 路由基线（当前）

- `#/` / `#/home`：首页
- `#/portal`：AI百科总览
- `#/list?tab=...`：分类列表
- `#/detail/:slug`：文章详情
- `#/term/:id`：术语详情
- `#/video/:slug`：视频详情
- `#/podcast/:slug`：播客详情
- `#/learning-path`：7天学习路径
- `#/search?q=...&tab=...`：搜索页
- `#/section/:key`、`#/tool/:id`：其它频道/工具

## 4. 当前已落地交互约定（关键）

### 4.1 列表页

- 顶部保留 6 个横向分类 Tab（字号偏小、不厚重）。
- 列表页内容区顶部有 `返回AI百科` 按钮（返回 `#/portal`）。
- 左侧内容，右侧侧栏（搜索 + 筛选 + 热搜）为主布局。
- 快讯支持两种展示：
  - 极简模式（更紧凑）
  - 详情模式（时间轴）
- 其它分类支持：
  - 列表视图
  - 卡片视图

### 4.2 详情页返回逻辑（重要）

- 文章详情通过 `from` 参数记录来源页。
- 返回按钮文案由来源自动计算：
  - 从总览进详情：`返回AI百科`
  - 从列表某 tab 进入：`返回AI快讯/返回深度好文/返回精选教程/返回术语百科/返回优质视频/返回AI播客`
- 文章详情面包屑中 `AI百科` 和所属模块可点击跳转到对应层级。

### 4.3 新标签打开规则（重要）

- 资讯文章详情（`#/detail/:slug`）统一使用新标签页打开。
- 术语详情（`#/term/:id`）仍为当前标签页跳转。
- 术语详情的上一条/下一条、相关术语会保留来源 `from`，保证返回逻辑一致。

### 4.4 总览页术语百科

- 术语卡片直接显示术语定义摘要。
- 已去掉“鼠标移入才出现解释浮层”的主交互模式。

### 4.5 播客详情

- 播客详情保留一套自定义进度条（可拖拽），作为主要播放进度交互。
- 避免双重进度体验（不再依赖显式原生 controls UI）。

## 5. 模块顺序（总览页当前版本）

`头图推荐+实时快讯` → `术语百科` → `优质视频` → `AI播客` → `深度好文` → `教程精选` → `推荐工具` → `订阅模块` → `为什么选择AI STORE`

补充：

- 深度好文 / 教程精选都支持“快捷标签”过滤。
- 教程筛选区旁会显示 `OpenClaw7天学习路径` 入口卡片（在列表教程 Tab 的侧栏中）。

## 6. 筛选与搜索（列表页）

- 右侧搜索框为全局入口（按当前 tab 搜索）。
- 筛选包含：分类/时间/来源/主题/排序。
- 术语百科 tab 不显示“来源”筛选（保留“主题”筛选）。
- 热搜卡点击会直接触发搜索并进入结果。

## 7. 视觉与风格共识（持续遵守）

- 主风格：深色、轻玻璃拟态、品牌绿高亮（`#1ed661`）。
- 内容卡片统一使用轻透明背景 + 低对比边框 + hover 提亮。
- 标题层级清晰，弱信息（时间、辅助文案）用灰阶降低视觉噪音。
- “查看更多/查看全部”使用统一文案和箭头语义，保持可扫描性。

## 8. 当前主要文件职责

- `src/App.tsx`
  - 核心路由、视图切换、筛选状态、详情交互与大部分业务逻辑都在此。
  - 包含关键导航工具函数：
    - `openNewsDetailInNewTab(...)`
    - `navigateToTermDetail(...)`
    - `getBackLabelFromHash(...)`
- `src/components/*`
  - 卡片与通用 UI。
- `src/data.ts` / `src/types.ts`
  - 模拟数据与类型定义。

## 9. 后续开发注意事项（避免回退）

- 新增“文章入口”时，默认使用 `openNewsDetailInNewTab`，并传入当前 hash 作为 `from`。
- 改动详情返回按钮时，不要写死为 `返回AI百科`，应继续走来源推导逻辑。
- 术语卡解释不要再改回 hover 浮层主模式。
- 播客详情避免再叠加重复进度条或重复播放器层。
- `App.tsx` 体量已很大，后续建议优先按页面拆分（列表页、详情页、术语页、播客页）。

## 10. 快速验收清单

- 列表页顶部存在 `返回AI百科`。
- 从列表进入文章详情，返回按钮文案与来源 tab 对应。
- 从总览进入文章详情，返回按钮为 `返回AI百科`。
- 总览术语卡直接显示定义描述，不依赖 hover 浮层。
- 播客详情只有一套主进度交互（可拖动）。
