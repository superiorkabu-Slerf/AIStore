# AIStore 资讯门户项目上下文

这份文档用于沉淀当前线程内已经明确的基础信息、命名定义和设计约定，后续继续完善项目时，优先以这里作为参考。

## 1. 项目定位

- 项目名称：`AIStore 资讯门户`
- 当前形态：单页应用（SPA），通过 `hash route` 在多个资讯/内容视图之间切换。
- 核心目标：把 AI 行业资讯、深度内容、教程、术语、视频、播客和学习路径整合到一个统一风格的内容门户里。
- 用户主路径：
  - 首页 `#/`
  - AI百科聚合页 `#/portal`
  - 分类列表页 `#/list?tab=...`
  - 详情页 `#/detail/:slug`
  - 术语详情 `#/term/:id`
  - 视频详情 `#/video/:slug`
  - 播客详情 `#/podcast/:slug`
  - 搜索页 `#/search?q=...&tab=...`
  - 7天学习路径 `#/learning-path`

## 2. 技术栈与运行方式

- 前端：React 19 + TypeScript
- 构建：Vite 6
- 样式：Tailwind 工具类 + `src/index.css` 中的手写全局样式
- 动效：`motion/react`
- 图标：`lucide-react`
- 当前主要命令：
  - 启动开发环境：`npm run dev`
  - 构建：`npm run build`
  - 类型检查：`npm run lint`

## 5. 内容分类定义

分类 `ListTab` 固定为以下 6 类：

- `flash`：AI快讯
- `article`：深度好文
- `tutorial`：精选教程
- `knowledge`：术语百科
- `video`：优质视频
- `podcast`：AI播客

这些分类是整个资讯系统的一级信息架构，后续新增模块时，除非有非常明确的产品理由，否则不要轻易改一级分类。

### 各分类的语义定义

- AI快讯
  - 时效性最强。
  - 阅读目标是“快速扫描重点事件”。
  - 标题优先级高于时间，时间应降级为辅助信息。

- 深度好文
  - 偏长文、分析、观察、解读。
  - 重点在摘要质量、标题辨识度、阅读入口清晰度。

- 精选教程
  - 偏实践、步骤、方法论、复用模板。
  - 重点在学习成本、难度标签、适合人群。

- 术语百科
  - 核心任务是“解释清楚概念”。
  - 术语解释应尽量直接展示在字段或卡片里，不要依赖悬浮浮窗才看得到定义。

- 优质视频
  - 重点在封面、时长、主题标签、信息密度。

- AI播客
  - 重点在标题、摘要、收听量、时长和播放体验。
  - 如果页面已经使用原生播放器，自定义进度展示不要重复堆叠。

## 7. 视觉系统基础定义

### 全局气质

- 整体方向：深色、玻璃拟态、轻微极光感背景、品牌绿高亮。
- 主品牌绿：`#1ed661`
- 学习路径模块可使用更亮一点的模块强调绿：`#00ff88`
- 基础背景应保持深炭黑/深蓝黑，不要做成纯平黑色大块压屏。

### 文本灰阶

后续写样式时，优先遵守 4 级灰阶逻辑：

- 一级：核心标题，接近纯白
- 二级：卡片标题、关键数字
- 三级：描述、次级说明
- 四级：图标、分隔线、弱提示

### 间距规则

- 信息模块之间要有明显留白，不要“组件贴组件”。
- 卡片间距、筛选区间距、列表区与侧栏区间距要统一。
- 如果出现“某块很挤，某块很空”，优先回到整体 spacing 规则统一处理，不要只补局部像素。

### 卡片规则

- 列表和学习路径等内容卡片，优先使用：
  - 轻透明背景
  - 轻模糊
  - 低对比边框
  - hover 时品牌绿边框/描边增强
- hover 动效以“轻微放大 + 边框提亮 + 内容渐显”为主，不做花哨位移动画。

### 图标规则

- 同一模块内图标保持线框风格一致。
- 默认使用暗灰/弱灰，激活或 hover 才转成品牌绿。

## 8. 当前线程里已经明确的设计共识

下面这些内容，可以看作本轮迭代中已经达成的“设计源规则”。即便个别页面还没有 100% 完成，也建议后续继续按这些规则收口。

### 资讯列表页

- 分类应放在顶部横排，不放右侧竖向分类块。
- 分类下方应有一条分割线。
- 分割线和选中项的绿色下划线要在同一高度上融合，不要上下错位。
- 列表页顶部的主标题/副标题应去掉，避免无效占高。
- 顶部筛选区域的模糊背景需要覆盖顶部和左右边缘，避免漏出后面的字。
- 筛选区域透明度不能太高，否则影响阅读。
- 整个资讯列表页的字号、颜色、边框、间距、背景透明度应统一，不要每个模块各用一套视觉规则。

### AI快讯

- 时间不是第一优先级，标题才是。
- 视觉上应弱化时间，强化标题扫描效率。

### 术语百科

- 术语解释应直接显示在内容字段里。
- 不要把“解释”主要藏到 hover 浮层里。

### 7天学习路径

- 需要玻璃拟态、轻透明、轻模糊、轻投影的卡片系统。
- 整个模块背景可以更浅一些，并加入渐变，而不是大块死黑。
- 顶部单独那块“背景框”应去掉或弱化，避免出现一整块厚重黑底。
- 模块内部强调“呼吸感”，需要更大的留白和卡片间距。
- 进入具体详情时，优先新标签页打开，保持学习路径总览不被打断。

### 详情页

- 详情页整体样式要和列表页统一到同一套设计语言中。
- 不同详情页之间的元信息、操作栏、卡片边框、留白、文字层级应尽量一致。

### 播客详情

- 如果页面上方已经有原生播放器进度，不要再重复堆一个几乎同义的自定义绿色进度条。
